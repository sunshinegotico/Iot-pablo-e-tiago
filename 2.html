HTTP (Hypertext Transfer Protocol)

O que é?

O protocolo fundamental da World Wide Web, adaptado para IoT através de APIs RESTful para permitir que dispositivos enviem dados diretamente para a nuvem.

Como funciona?

Baseia-se no modelo Requisição-Resposta. O dispositivo (cliente) solicita algo e o servidor processa e responde. Utiliza métodos como GET (leitura) e POST (envio de dados).

Características Técnicas

Atributo Descrição
Arquitetura Cliente-Servidor (Stateless)
Segurança HTTPS (TLS/SSL)
Payload Geralmente JSON ou XML
Consumo Alto (devido ao overhead do cabeçalho)
Vantagens: Onipresente, fácil de implementar em qualquer linguagem e possui segurança nativa robusta.
Desvantagens: Consumo elevado de memória e bateria; não é ideal para transmissões em tempo real constante.
